# 📱 คู่มือการพัฒนา Income Tracker (Flutter Guide)

รวบรวมคำสั่งที่จำเป็นสำหรับการพัฒนาและติดตั้งแอปบน iPhone/Simulator แบบเข้าใจง่ายครับ

---

### 🚀 1. การรันแอปเพื่อพัฒนา (Development)
ใช้ตอนที่คุณกำลังแก้โค้ดและต้องการเห็นผลทันที (Hot Reload)

*   **เปิดเครื่องจำลอง (iOS Simulator):**
    ```bash
    open -a Simulator
    ```
*   **สั่งรันแอป (โหมดพัฒนา):**
    ```bash
    flutter run
    ```
*   **ปุ่มลัดขณะเครื่องกำลังรัน (ใน Terminal):**
    *   `r` : **Hot Reload** (อัปเดตหน้าจอทันทีหลังจากเซฟไฟล์) - *เร็วมาก*
    *   `R` : **Hot Restart** (เริ่มแอปใหม่แต่ไม่ปิดแอป) - *ใช้ตอนแก้ไฟล์ที่มีผลต่อโครงสร้างแอป*
    *   `q` : **Quit** (ปิดแอปและหยุดการรัน)

---

### 💎 2. การติดตั้งลง iPhone เครื่องจริง (Deployment)
ใช้เพื่อติดตั้งแอปให้เสถียรที่สุดและถือออกไปใช้งานข้างนอกได้ (อยู่นาน 7 วัน)

*   **รันแบบใช้งานจริง (Release Mode):**
    ```bash
    flutter run --release
    ```
    *(แนะนำให้เสียบสาย USB และลบแอปเก่าออกก่อน เพื่อความสะอาดที่สุด)*

---

### 🛠️ 3. การแก้ไขเมื่อเกิดปัญหา (Troubleshooting)
ถ้าแอปเด้ง รันไม่ติด หรือขึ้น Error แปลกๆ ให้รันชุดคำสั่ง "ล้างระบบ" นี้ครับ

1. ล้าง Cache ทั้งหมด:
   ```bash
   flutter clean
   ```
2. โหลด Library ใหม่:
   ```bash
   flutter pub get
   ```
3. ล้างระบบฝั่ง iOS (ถ้าติดเรื่อง iOS/iPhone):
   ```bash
   cd ios && rm -rf Pods Podfile.lock && pod install && cd ..
   ```

---

### 🛡️ 4. ข้อควรรู้เรื่องใบรับรอง Apple (7 วัน)
แอปที่ลงเอง (Side-loading) สำหรับ Apple ID ทั่วไปจะมีอายุ **7 วัน**

*   **เมื่อครบ 7 วัน:** แอปจะเปิดไม่ติด (เด้งออก)
*   **วิธีแก้:** เสียบสายกับ Mac แล้วรัน `flutter run --release` อีกครั้ง (ข้อมูลรายได้จะไม่หาย)
*   **เปิดโหมดนักพัฒนา:** หากเป็นเครื่องใหม่ ให้ไปที่ *Settings > Privacy & Security > Developer Mode > On*

---

### 📝 5. คำสั่งเช็คระบบอื่นๆ
*   `flutter doctor` : ตรวจสอบความพร้อมของ Flutter ในเครื่อง
*   `flutter devices` : ดูรายชื่อมือถือ/Simulator ที่เชื่อมต่ออยู่
*   `flutter analyze` : ตรวจสอบว่าโค้ดมีตรงไหนเขียนผิดหลักการไหม
